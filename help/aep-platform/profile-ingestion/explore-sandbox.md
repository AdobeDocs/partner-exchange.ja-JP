# AEP サンドボックスへのアクセスと探索

この記事では、次の内容について説明します。

* 既存のAdobe Exchangeパートナーサンドボックス組織と共有 AEP サンドボックスの違い
* AEP 共有サンドボックスへのアクセスをリクエストしています。
* AEP 共有サンドボックスへの招待メールを受信します。
* [!DNL Admin Console] に新規ユーザーを招待しています。
* AEP UI の操作。

AEP のサンドボックステクノロジーの一般的な概要については、この [&#x200B; 記事 &#x200B;](https://docs.adobe.com/content/help/ja-JP/experience-platform/sandbox/home.html) を参照してください。

## 共有 AEP サンドボックス

Exchange パートナーには、独自のAdobe[!DNL Experience Cloud] 組織（非共有）を通じて、様々なAdobe[!DNL Experience Cloud] 製品（[!DNL Analytics]、[!DNL Target]、Platform タグなどの非 AEP 製品）へのアクセス権が付与されます。 パートナーには、ユーザーやその他の権限を管理する組織のシステム管理者アクセス権が付与されます。 Adobe[!DNL Experience Platform] （AEP）の扱いは、他のAdobeサンドボックスとは異なります。 主な違いは次のとおりです。

* AEP へのアクセスは、パートナーのメインAdobe[!DNL Experience Cloud] サンドボックス組織を通じて行われません。
* AEP には、共有Adobe Exchange組織を通じてアクセスします。
* 他の多くのAdobe Exchangeパートナー会社も同じ組織を使用して AEP にアクセスしています
   * AEP サンドボックス機能により、この共有組織内のデータとアクティビティを他のパートナーが表示または変更することはできません。各パートナーは、共有組織内の別のサンドボックスにアクセスできるようになります。
* この共有組織内の管理権限は非常に制限されています。
* AEP のサンドボックスへのアクセス権が付与されると、パートナーには、Admin ConsoleまたはメインExperience Cloudのホームページで、UI の右上にある組織スイッチャーに 2 つの組織が表示されます。 ただし、AEP にログインした場合は、共有組織のみ表示されます。

## 共有 AEP サンドボックスへのアクセスをリクエスト

次の情報を入力して [&#x200B; サポートリクエスト &#x200B;](https://adobeexchangeec.zendesk.com/hc/ja-jp/requests/new) を送信します。

* メールアドレス
* 件名：AEP サンドボックスリクエスト
* 製品：一般プロビジョニング/サンドボックス
* チケットの種類：プログラムのサポート - Exchange プログラム/プロビジョニングリクエストに関する質問
* 説明：AEP サンドボックスの使用が必要な統合ユースケースの簡単な説明を提供します
* また、AEP サンドボックスに追加する必要があるすべてのユーザー名とメールを必ず指定してください。 リクエスト後にユーザーを追加することもできますが、Adobeが追加のチケットを使用してユーザーを追加する必要があります（以下を参照）。

## E メールによる招待状を受信

AEP サンドボックスをリクエストしたプライマリ連絡先には、Adobe[!DNL Experience Platform] を「開始」するよう招待する自動メールが届きます。 また、プライマリ連絡先にはいくつかの管理権限もあります。これらについては、次の節で説明します。

招待状のメールアドレスに関連付けられているAdobe IDで `https://platform.adobe.com.` ログインに直接移動するか、Adobe IDに関連付けられていない場合は作成してください。

## 他のユーザーを招待

次の情報を入力して [&#x200B; サポートリクエスト &#x200B;](https://adobeexchangeec.zendesk.com/hc/ja-jp/requests/new) を送信します。

* 要求者の E メール アドレス
* 件名：AEP サンドボックス – 管理者/ユーザーを追加
* 製品：一般プロビジョニング/サンドボックス
* チケットの種類：プログラムのサポート - Exchange プログラム/プロビジョニングリクエストに関する質問
* 説明：追加するユーザー（名前とメール）のリスト

## AEP UI の操作

AEP UI を見る [&#x200B; 概要ビデオ &#x200B;](https://docs.adobe.com/content/help/ja-JP/platform-learn/tutorials/intro-to-platform/interface-tour.html)

AEP UI には、左側のパネルを使用して移動できる 12 の主な領域があります。 ただし、このタイプの統合で最も重要なセクションは、スキーマ、データセット、プロファイルです。

* ホーム – ランディング画面

   * いくつかの「はじめに」アクティビティを提案します
   * 学習コンテンツへのリンクがいくつか表示されます
   * スキーマ、データセット、プロファイルなど、主な AEP オブジェクトの一部をダッシュボードに表示します

* ワークフロー – データを AEP に取り込むための一般的なワークフローの開始
* 接続/ソース - AEP に取り込まれるデータのソースを管理します
* 接続/宛先 – 外部システムにデータを送信するための接続を管理します
* プロファイル – 個々の顧客プロファイルの表示と管理
* セグメント – 顧客セグメントを参照、作成および変更します。
* ID - ID 名前空間を参照、作成および変更します。これらは、顧客を一意に識別するために使用されるプライマリ ID のタイプです
* モデル（データサイエンス） – 埋め込み Jupyter Notebooks 環境の使用など、データサイエンス活動に従事します。
* サービス（データサイエンス） – データサイエンスのレシピをサービスとして公開します
* スキーマ – スキーマを参照、作成、変更します。これらは、データを整理するための詳細なデータ定義です
* データセット – データセットの参照、作成、管理。データセットは、スキーマによって定義され、AEP 内でデータが存在する場所になります
* クエリ – クエリのリポジトリを参照、作成、変更および使用して、データセット内のデータからインサイトを得ます
* 監視 – AEP に出入りするすべてのデータのステータスを、バッチとストリーミングの両方で表示できます
